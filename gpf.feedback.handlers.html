gpf.feedback.handlers module¶

This module contains Handlers for the ArcLogger and Logger classes.

class gpf.feedback.handlers.FileLogHandler(filename, time_tag=True, encoding='cp1252')[source]¶

Bases: logging.handlers.RotatingFileHandler

Custom file handler that inherits from RotatingFileHandler.

By default, the preferred system encoding is used (often set to CP1252 or UTF-8).

When a gpf.feedback.loggers.Logger or gpf.feedback.loggers.ArcLogger is set up with this file handler, new logger instances (e.g. in other modules or script) will use the same file handler if these loggers are initialized using the same filename.

Parameters:
  • filename (str) – Log file name or absolute or relative path to the log file. If filename does not have an extension, a .log extension will be added. If filename has an extension that does not equal .txt or .log, it will be set to .log. If time_tag is True (default), a _YYMMDD_HHMMSS timestamp (current local time) will be added to the log file name automatically. If filename is just a name, the output directory will be set to the user temp directory. If filename is a relative path, it will be made absolute (relative to os.curdir). If filename is an absolute path, that path will be used as-is. If the log directory of filename does not exist, it will be created.
  • time_tag (bool) – If set to True (default), a _YYMMDD_HHMMSS timestamp will be appended to filename. When set to False, filename will not receive a timestamp.
  • encoding (str) – The optional encoding to use for the output file. Defaults to the preferred system encoding.

Warning

The user must have write access to the specified output directory.

identity¶

Returns the “identity” of the FileLogHandler, which is the name or path that was used to instantiate it.

Note that this often does not equal the path which the FileLogHandler is actually writing to.

class gpf.feedback.handlers.ArcLogHandler(stream=None)[source]¶

Bases: logging.StreamHandler

Custom log handler that writes to the standard stream (e.g. console) when the log level >= _logging.DEBUG. The handler also sends messages to ArcGIS when the log level >= _logging.INFO.

emit(record)[source]¶

Emit a record. Override of the original StreamHandler.emit() -> see _logging.StreamHandler for documentation. Writes messages to ArcGIS for all log levels defined in __FUNC_MAP. Writes to stderr when the log level has been set to DEBUG.
